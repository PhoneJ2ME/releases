The CDCAms AppManager allows for a modular, object-oriented design. Each functional area is clearly defined via a set of Java interfaces. Customers can replace the presentation layer of the AppManager to allow for different user interaction models. A presentation layer can assume the existence of objects implementing the functional interfaces.

This document outlines the public interfaces for the modules of AppManager. Each module resides in its own package, and consists of a set of Java classes and interfaces. Instances of each module can be accessed via the {@link com.sun.appmanager.AppManager} class.
